# Verify Build

Проверка проекта через сборку. Это единственный способ валидации — тестов нет.

## Команда

```bash
npm run build
```

(под капотом: `tsc && vite build`)

## Что проверяет

- TypeScript компиляция: типы, `noUnusedLocals`, `noUnusedParameters`, strict
- Vite bundling: import resolution, asset handling
- Все модули резолвятся корректно

## Критерии успеха

- Exit code 0
- В output нет `error TS...` или `[vite]: Rollup failed`
- Warning про chunk size (three.js большой) — допустим, игнорировать

## Когда запускать

- После КАЖДОГО завершённого изменения
- Перед коммитом
- Перед тем как сообщить пользователю что задача выполнена

## Если билд упал

1. Прочитать ошибку
2. Исправить корневую причину (не глушить через `any`/`@ts-ignore` без серьёзных оснований)
3. Перезапустить `npm run build`
4. Не сообщать об успехе пока билд не пройдёт

## После билда

- Если задача в `.claude/tasks/{name}/` не завершена — обнови `progress.md`
- Если задача завершена — поставь статус `done` в `progress.md`

## Ручная проверка UI

Билд не проверяет, что игра реально работает. Если правил визуал/ввод/физику — запусти `npm run dev` и проверь руками:
- Кости видны и подсвечены
- ЛКМ + движение мыши = кости тянутся за курсором с тряской
- Отпускание = кости летят и катятся, потом останавливаются
- Resize окна не ломает рендер
